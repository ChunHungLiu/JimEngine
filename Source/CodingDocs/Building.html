Setting up and building the engine

Setting up the source and building the engine is a bit quirky. At the moment (potentially subject to change if it turns out to be too annoying), it's set up for a specific directory setup. At the root directory you have things set up in, there should be two directories, Source and Build. Source should contain all the projects, and Build should contains the main game binaries and game data. Example:

[Root Directory]
    >   Source
        >   AmimSeq           --animation sequencer plugin
        >   Editor            --editor
        >   Engine            --engine
        >   XenLib            --static library for plugins
        >   XenModelExporter  --model exporter
        >   CodingDocs        --coding documentations

    >   Build                 --contains main game binaries, configs, etc
        >   Data              --contains all the game data

It's important to have this directory set up because when you build or rebuild any of your files it's configured to automatically copy the files to the Build when you build them.

Compiling

If you're compiling for the first time, you'll want to first build the OGG sdk. Load Source\Engine\oggsdk\Waaaogg.sln and build releasestatic, debugstatic, and their respective 64bit versions. After doing that you should never need to build any of them again.

After that, you'll then want to build the Source\Engine\Engine.sln project, then Source\Editor\Editor.sln, then Source\[MyGameModule]\[MyGameModule].sln.

Whenever you rebuild these projects they will automatically copy over to the build directory for you after being built.

Setting it up for debugging

Because debug locations and such are local to each user and not stored in the main project files, each user must set their own respective debug locations in their project settings. For Engine.sln, you must do this in the Game sub-project. So you'd open Engine.sln, right click the Game project, goto properties, then in the Debug section, you'll want to set your debug directory to your Build path. You will also have to do this for the Editor project in the Editor solution.

For the main game module project, you will have to set both the debug directory as well as set it to execute Game.exe. After that you should be ready to go.
