Frame Objects (Object.h)

class BASE_EXPORT FrameObject : public Object

Using this class allows you to take advantage of certain implementable functions that are called each frame for all objects created that are derived of this class.

All 2D (Window) and 3D (Entity) objects are derived from FrameObject.

Any class derived from FrameObject will default to non-renderable (unless derived from a renderable class). To set that object as a renderable object, then, for example, you would do this:

class IAmMadeOf3DCheese : public Entity
{
    CLASS_DECLARE(IAmMadeOf3DCheese, Entity);
    RENDERABLE(TRUE);
    ...
};

SafeDestroy

This will make a frame object automatically destroy itself next frame. Very useful in things like particle systems where you might want some object to automatically destroy itself safely.

myExampleObject->SafeDestroy();

FrameObject Implementable functions

virtual void PreFrame()

Called once right before the frame begins

Unless purposely trying to bypass a PreFrame() of a superclass always call Super::PreFrame();.

virtual void Tick(DWORD dwMS, float fSeconds)

Called once per frame after PreFrame for each existing FrameObject that exists.

Unless purposely trying to bypass a Tick() of a superclass always call Super::Tick(dwTime, fSeconds);.
