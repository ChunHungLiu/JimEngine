GameModule class (GameModule.h)

This class is used as the entry point for the game inside a game module. For the main game module, it must be derived from this class and named MainGameModule.

Then to load your module, you would use the game ini file and set the GameModule value in [Engine] to the name of your game module. So for example if I had a game module called "Hyperion.xxt", I would use "GameModule=Hyperion" in the [Engine] section, and then when the game loads it would automatically create and call the functions of the MainGameModule class within that module.

GameModule implementable functions

class BASE_EXPORT GameModule : public Object
virtual BOOL GameStartup(TSTR lpParameters, int numParameters)=0;

GameStartup is called after the engine is initialized, right before the main engine loop begins.

lpParameters contains the parameters used when the application started. numParameters specifies the number of parameters.

Typically this is where you create things like your main menu and such when the game starts up.

virtual void GameExit()=0;

GameExit is called right after the main engine loop exits, and right before the main termination of all engine objects.
